## AWS Global Infrastructure
###### AWS Responsibility - Resiliency of the cloud
- AWS is responsible for resiliency of the Infrastructure that runs all of the services offered in the AWS Cloud.
###### Customer Responsibility - Resiliency in the cloud
- is determined by the AWS cloud services that you select

#### Region
- AWS Region은 다중의 가용 영역으로 구성된다. (AZ, AWS Availability Zone)
	- [AWS Availability Zones](https://docs.aws.amazon.com/global-infrastructure/latest/regions/aws-availability-zones.html)
		- Each Region has at least three Availability Zones. This helps you to design highly available applications on AWS.
- 데이터 센터를 클러스터링하는 물리적 위치
<img width="2172" height="666" alt="Pasted image 20251230175633" src="https://github.com/user-attachments/assets/e6c578bb-0345-4443-9093-06da1e9cb2cd" />

- [AWS Fault Isolation Boundaries](https://docs.aws.amazon.com/whitepapers/latest/aws-fault-isolation-boundaries/abstract-and-introduction.html)
#### Availability Zone
- 애플리케이션과 데이터는 다른 AZ 간에 실시간으로 복제되며 일관성을 가진다.
- AWS Region의 중복 전력, 네트워킹 및 연결이 제공되는 하나 이상의 개별 데이터 센터로 구성된다.
- The geography for an Availability Zone is the specific physical location of its infrastructure.
	- [AWS Availability Zones](https://docs.aws.amazon.com/global-infrastructure/latest/regions/aws-availability-zones.html)
## AWS Compute
### Amazon EC2
- 클라우드에 위치한 Virtual Server Instance
- [AWS Documentation - What is Amazon EC2?](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/concepts.html)
	- Amazon Elastic Compute Cloud (Amazon EC2) provides **on-demand**, **scalable** computing capacity in the AWS Cloud.
	- You can use Amazon EC2 to launch **as many or as few virtual servers** as you need, configure security and networking, and manage storage.
	- add capacity to handle compute-heavy tasks or spikes in website traffic (Scale Up)
	- reduce capacity when usage decreases (Scale Down)

### Storage Options for Amazon EC2 instances

#### Block Storage
- 데이터를 일정 크기의 블록으로 나누어 저장
- 호스트에서 파일 시스템을 생성
- Storage Area Network(SAN)
##### Block Storage: Amazon EBS
- durable, block-level storage volumes
- can attach and detach from your instances
- can attach multiple EBS volumes to an instance.
- An EBS volume persists independently from the life of its associated instance.
- can create snapshots (that are stored in Amazon S3).

#### File Storage
- 디렉토리 구조로 파일 저장
- 스토리지 레벨에서 파일 시스템을 생성
- Network Attached Storage(NAS)
##### File Storage: Amazon EFS
[AWS Documentation - How Amazon EFS works](https://docs.aws.amazon.com/efs/latest/ug/how-it-works.html)
<img width="1120" height="823" alt="Pasted image 20260104155906" src="https://github.com/user-attachments/assets/2798f94f-9dc8-4ebf-8117-3e4196b07fe1" />

- provide scalable file storage for use with Amazon EC2.
	- simple, serverless, set-and-forget elastic file system.
- create an EFS file system and configure your instances **to mount the file system**.
	- mount an EFS file system in your **virtual private cloud(VPC)**
- use EFS as a common data source for workloads and applications running on multiple instances.
##### File Storage: Amazon FSx
- launch, run, and scale feature-rich, high-performance file system.
- fully-managed service that supports a wide range of workloads.
