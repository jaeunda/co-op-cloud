## Cloud Native

![[Pasted image 20260104170705.png]]
- 클라우드 환경에 최적화된 애플리케이션 설계 방식
- 구성: DevOps, CD, Microservices, Container
- 특징
	- 경량화
	- 자동 확장
	- 장애를 전제로 설계
## Microservice Architecture
![[Pasted image 20260104170833.png]]
- 하나의 큰 애플리케이션을 작은 서비스 단위로 분리
- 장애 격리가 가능하며 확장이 유연하다는 장점
## Container
![[Pasted image 20260104170952.png]]
- 애플리케이션 + 실행 환경을 하나로 묶은 실행 단위
- OS를 포함하는 VM(Hypervisor)과 달리 컨테이너는 Host OS를 공유
## DevOps
![[Pasted image 20260104171006.png]]
- 개발(Dev) + 운영(Ops)을 하나의 흐름으로 통합
