###### Amazon EKS
![[Pasted image 20260104184143.png]]
- managed service that you can use to run Kubernetes on AWS **without needing to install, operate, and maintain your own Kubernetes control plane or nodes**.
- high availability by running Kubernetes control plane instances across multiple AZs.
##### Install `kubectl` && `eksctl`
- `kubectl` CLI를 사용하면 해당 명령어가 Kubernetes API를 호출해 관련 동작을 수행한다.
- `eksctl` CLI를 사용하면 EKS 클러스터를 생성 및 관리할 수 있다.
![[Screenshot 2025-12-23 143224.png]]
##### Build Container Image
1. `Dockerfile` 작성: 컨테이너 이미지를 만들기 위한 설정 파일
2. `docker build`: 이미지 생성
![[Screenshot 2025-12-23 143236.png]]
###### Instruction Component in the Dockerfile
![[Pasted image 20260104184713.png]]

![[Screenshot 2025-12-23 143300.png]]
3. `docker images`: 생성된 이미지 확인
4. `docker run`: 이미지를 컨테이너로 실행
	- `test-image`라는 컨테이너 이미지를 사용하여 
	- `test-nginx`라는 이름의 컨테이너를 실행하고
	- 호스트 `8080` 포트와 컨테이너의 `80` 포트 매핑(도커를 통해 포워딩)

![[Screenshot 2025-12-23 143630.png]]
##### Upload container image to Amazon ECR
1. 이미지를 빌드한 후 `docker tag`: 해당 이미지가 특정 repository에 푸쉬될 수 있도록 설정
2. `docker push`: 이미지를 repository에 푸쉬
